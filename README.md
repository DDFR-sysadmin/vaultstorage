# 🔐 VaultStorage

**VaultStorage** — это легковесная CLI-утилита для безопасного локального хранения чувствительных данных (токенов, ключей, API) в формате JSON.

## 🎯 Мотивация

Утилита создавалась для решения классической проблемы автоматизации: **как безопасно передать секреты в скрипты, выполняемые через `cron`?** Хранить чувствительные данные (API-ключи, пароли, токены) прямо в теле скрипта — небезопасная практика, так как скрипты могут случайно попасть в системы контроля версий или быть прочитаны другими пользователями. VaultStorage решает эту задачу, предоставляя интуитивно понятный интерфейс командной строки для безопасного хранения и точечного извлечения данных из изолированных JSON-хранилищ.

## 🛡️ Безопасность и архитектура

Защита данных построена на строгом разграничении прав доступа на уровне файловой системы Linux:

* **Изоляция пользователей:** Утилита проверяет, создает и изменяет файлы только с правами **того пользователя, от имени которого она запущена**.
* **Защита от Privilege Escalation (PE):** Директория с секретами (`/opt/vaultstorage/`) использует Sticky Bit. Любой другой пользователь системы (без прав `root` или `sudo`) не сможет получить доступ к чужим хранилищам или модифицировать их.
* **Строгие права доступа:** Файлам секретов жестко задаются права `600` (чтение/запись только для владельца), а персональным директориям — `700`.

## 🚀 Установка

Для установки утилиты в систему перейдите в директорию с исходными файлами и выполните установочный скрипт с правами суперпользователя:

```bash
chmod +x install.sh && sudo ./install.sh

```

*Скрипт создаст защищенную базовую директорию `/opt/vaultstorage`, скопирует утилиту в `/usr/bin/vaultstorage` и сделает её исполняемой.*

**Деинсталляция:**
Если утилиту потребуется удалить, просто выполните:

```bash
sudo ./uninstall.sh

```

## 💻 Использование

Синтаксис предельно простой:

```bash
vaultstorage [Название хранилища] [флаги]

```

*(Если вы передаете только название хранилища без флагов, утилита выведет весь JSON-объект в консоль).*

### Доступные флаги

| Флаг | Описание | Пример использования |
| --- | --- | --- |
| `-i` | **Извлечь данные.** Получить значения по одному или нескольким ключам (через пробел или запятую). | `vaultstorage google -i API token` |
| `-A` | **Добавить пару.** Добавить или обновить одну запись в формате `ключ:значение`. | `vaultstorage google -A "token:AIzaSyD..."` |
| `-a` | **Добавить из файла.** Дописать или обновить данные из указанного локального JSON-файла. | `vaultstorage google -a ./new_keys.json` |
| `-E` | **Перезаписать парой.** Полностью перезаписать хранилище одной парой `ключ:значение`. | `vaultstorage google -E "token:12345"` |
| `-e` | **Перезаписать из файла.** Полностью перезаписать хранилище данными из указанного JSON-файла. | `vaultstorage google -e ./backup.json` |
| `-sF`, `--show-files` | **Список хранилищ.** Показать все доступные хранилища текущего пользователя. | `vaultstorage -sF` |
| `-sS`, `--show-secrets` | **Просмотр секретов.** Вывести содержимое конкретного или всех хранилищ в удобном читаемом виде. | `vaultstorage google -sS` или `vaultstorage -sS` |

## 📖 Примеры сценариев

**1. Создание хранилища и добавление данных:**

```bash
# Создаст хранилище "aws_prod" (или обновит, если существует) и добавит ключи
vaultstorage aws_prod -A "access_key:AKIAIOSFODNN7EXAMPLE"
vaultstorage aws_prod -A "secret_key:wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"

```

**2. Использование в Bash/Cron скрипте:**
Вместо хардкода секретов в bash-скрипте, просто вызовите утилиту:

```bash
#!/bin/bash

# Безопасно вытаскиваем токен прямо в переменную
TG_TOKEN=$(vaultstorage telegram -i token)

# Используем токен для запроса
curl -s -X POST "https://api.telegram.org/bot${TG_TOKEN}/sendMessage" \
    -d chat_id="123456789" \
    -d text="Бэкап базы данных успешно завершен!"

```

**3. Быстрая проверка своих секретов:**

```bash
# Узнать, какие хранилища у вас есть
vaultstorage -sF

# Посмотреть содержимое всех своих хранилищ
vaultstorage -sS

```
